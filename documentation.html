1. Introduction

Weaveries is a WordPress plugin for personal knowledge management (PKM), worldbuilding, and project management. It lets you create structured, interconnected notes called Weaves — each with custom types, statuses, fields, and hierarchical relationships — and display them anywhere on your site using shortcodes.

Key capabilities:

  • Create and organise Weaves in a drag-and-drop tree (the Outliner)
  • Assign Types with custom icons and colours
  • Track progress with configurable Status fields (Kanban-style)
  • Attach Custom Fields (text, number, date, dropdown, checkbox, …)
  • Link Weaves together with named Relationship types
  • Embed Kanban boards, calendars, lists, and outliners on any page
  • Use the full-screen Standalone Editor for a distraction-free writing experience

2. Core Concepts

Weaves

A Weave is the fundamental unit of content — equivalent to a WordPress post of the custom type weave. Each Weave has a title, a rich-text body (powered by the WordPress block editor or the standalone editor), and can be connected to any number of other Weaves through the Outliner and Relationships systems.

Weaves can have one of the following WordPress post statuses:

  • Published — visible to all visitors according to your theme.
  • Draft — visible only to logged-in editors.
  • Private — visible only to logged-in users.
  • Archived — a custom status that hides a Weave from Kanban boards and normal listings. It can be restored at any time.

3. Types

Types categorise your Weaves. You might have types like Task, Note, Character, Location, or Project. Each type has a name, slug, colour, and optional Font Awesome icon.

Types are managed under Weaves → Settings → Types.

System Types

The plugin seeds a small set of built-in system types that cannot be deleted:

  • Note — general-purpose note (blue, fa-solid fa-note-sticky)
  • Task — actionable item (green, fa-solid fa-check-circle)
  • Category — grouping container (yellow, fa-solid fa-folder-open)

You can still customise the colour and icon of system types, and create as many additional types as you need.

Assigning Types to a Weave

Types are assigned in the Weave's edit screen in the WordPress admin, or inline on the frontend using the [weave types edit="true"] shortcode. A Weave can have multiple types simultaneously.

4. Statuses

Statuses let you track the state of a Weave through a user-defined workflow. A Status Field (e.g. Task Status) has a slug and a list of Status Options (e.g. To Do, In Progress, Done), each with a configurable colour.

Status Fields are managed under Weaves → Settings → Statuses.

Assigning Status Fields to Types

Each type can have one or more status fields assigned to it. One field per type is marked as the main field, which is used by Kanban boards and the auto-resolving status shortcodes. You can also designate a globally-default status field that is preferred when a Weave has multiple types.

Setting a Status Value

Status values are set on the Weave's edit screen, or interactively on the frontend using the [weave status picker="true"] shortcode.

5. Custom Fields

Custom Fields let you attach structured data to Weaves. Fields are created under Weaves → Settings → Fields and can optionally be scoped to specific Types.

6. Relationships

Relationships let you link two Weaves together through a named connection type — for example References, Inspired By, or Part Of. Relationship Types are managed under Weaves → Settings → Relationships.

Relationships are directional: Weave A references Weave B. You can view and edit all relationships attached to a Weave from the Relationships meta box in the edit screen.

7. The Outliner

The Outliner is the hierarchical tree view of all your Weaves. Parent-child relationships here are separate from Relationship types — they represent structural containment (e.g. a Project Weave that contains Task Weaves).

Key features

  • Drag and drop — reorder and reparent Weaves by dragging. When dropping onto an existing parent, a modal lets you choose between Move here (single parent) and Add as extra parent (multi-parent).
  • Multi-parent support — a Weave can appear under more than one parent simultaneously.
  • Type icons and colours — each Weave row shows its assigned type icons colour-coded by the type's colour.
  • Collapse/expand — click the arrow next to any Weave to collapse its subtree.
  • Auto-open — when the Outliner is shown on a Weave's own page, the tree automatically opens and highlights the current Weave.

The Outliner is embedded using the [weave outliner] shortcode, or shown in the left panel of the Standalone Editor.

8. The Standalone Editor

The Standalone Editor is a full-screen, three-pane writing environment accessible from Weaves → Open Editor in the WordPress admin, or directly via /?weaveries_editor=1.

Append &post_id=123 to open a specific Weave. Requires the user to be logged in with edit_posts capability.

Layout

  • Left pane — Outliner tree (searchable) and a Search tab to quickly find any Weave by title, type, or status.
  • Centre pane — Title input and the WordPress block editor for the Weave's content.
  • Right pane — Details panel (status, type, dates) and a Relations tab.

All three panes can be resized by dragging the dividers. The left and right panes can be collapsed with the chevron buttons. Panel widths and active tabs are saved per post in localStorage.

Hamburger menu

The ☰ button in the top-left opens a slide-down menu for quick navigation links (content coming in a future release).

9. Shortcodes

All shortcodes use the tag [weave …]. The first word after weave (written as a valueless attribute) selects the component. An additional helper shortcode [weaveries_ref] creates inline links.

Tip: Shortcodes that show editable controls (drag-and-drop, status pickers, type tags, etc.) only render their editing UI for logged-in users with the edit_posts capability. Visitors always see the read-only version.

9.1 Display Shortcodes

[weave field="field_name"]

Outputs the value of a custom field for the current Weave (or a specific one with id="X").

Built-in field names: title, content, excerpt.

AttributeDefaultDescription
fieldrequiredThe field_name slug of the custom field, or a built-in name (title, content, excerpt).
idcurrent postID of the Weave to read from.
 
[weave field="title"]
[weave field="due_date"]
[weave field="content" id="42"]

[weave status] / [weave status="slug"]

Displays the current status of a Weave as a styled coloured badge. Without a slug it auto-resolves the main status field for the Weave's assigned type.

AttributeDefaultDescription
statusauto-resolvedSlug of the status field to show.
idcurrent postID of the Weave to read from.
 
[weave status]
[weave status="task-status"]
[weave status="task-status" id="42"]

[weave status picker="true"]

Displays all status options as clickable coloured pills. Clicking a pill saves the new status instantly via AJAX (editors only; read-only for visitors).

AttributeDefaultDescription
statusauto-resolvedSlug of the status field.
pickerfalseSet to true to enable the interactive picker.
idcurrent postID of the Weave to read/write.
 
[weave status picker="true"]
[weave status="task-status" picker="true"]

[weave types]

Shows the types assigned to the current Weave as coloured pills. With edit="true", logged-in editors can add and remove types via an autocomplete tag input. Changes are saved instantly via AJAX.

AttributeDefaultDescription
editfalseSet to true to enable the editable tag input (editors only).
idcurrent postID of the Weave to show/edit types for.
 
[weave types]
[weave types edit="true"]

[weaveries_ref id="X"]Label[/weaveries_ref]

Creates a hyperlink to a specific Weave by its ID. The link text defaults to the Weave's title if no inner content is given. If the Weave is deleted, the shortcode gracefully falls back to plain text.

[weaveries_ref id="42"]My Weave[/weaveries_ref]
[weaveries_ref id="42"][/weaveries_ref]

9.2 Navigation & Hierarchy Shortcodes

[weave outliner]

Embeds the full hierarchical Outliner tree. The tree auto-expands to the current Weave when placed on a Weave singular page. With edit="true", logged-in editors can drag and drop to restructure the hierarchy.

AttributeDefaultDescription
editfalseEnable drag-and-drop reordering (editors only).
viewemptyOptional view parameter passed to Weave links (for theme templates).
 
[weave outliner]
[weave outliner edit="true"]
Multiple [weave outliner] shortcodes can appear on the same page — each runs as a fully independent instance.

[weave hierarchy]

Shows two sections — Parents and Children — listing all Weaves directly above or below the current Weave in the Outliner hierarchy. With edit="true", editors can add or remove parents and children via an autocomplete tag input saved via AJAX.

AttributeDefaultDescription
editfalseEnable interactive parent/child editing (editors only).
idcurrent postID of the Weave to show hierarchy for.
 
[weave hierarchy]
[weave hierarchy edit="true"]

[weave favourites]

Displays a horizontal bar of all Weaves marked as favourites, as clickable pills. With edit="true", editors can add or remove favourites directly via autocomplete. Syncs live with the star button in [weave toolbar] on the same page.

AttributeDefaultDescription
editfalseEnable the editable autocomplete input (editors only).
 
[weave favourites]
[weave favourites edit="true"]

9.3 Kanban Shortcode

[weave kanban]

Embeds a Kanban board grouped by the options of a status field. Each column represents one status option; Weaves appear as cards inside the matching column. Weaves with no status value appear in an Unassigned column on the left.

AttributeDefaultDescription
statusauto-resolvedSlug of the status field to group by. If omitted, the main status field of the current Weave's type is used.
scopeallall — show every Weave with this status type.
children — restrict to the current Weave's full descendant subtree.
editfalseEnable drag-and-drop between columns to reassign status (editors only).
idcurrent postWhen scope="children", the root Weave ID. Defaults to the current post.
 
[weave kanban]
[weave kanban status="task-status"]
[weave kanban status="task-status" scope="children" edit="true"]

Lane collapse: Each Kanban lane has a collapse button. Collapsed/expanded state is saved per status field and root post in localStorage, so it persists across page loads.

9.4 Calendar Shortcode

[weave calendar]

Embeds a full FullCalendar view. Weaves with a date field can be scheduled and displayed on the calendar. Logged-in editors can create new Weaves or assign existing ones to a date by clicking/dragging on the calendar. Events can be moved by dragging and removed via a right-click context menu.

AttributeDefaultDescription
outlinerfalseSet to true to show the Outliner tree as a collapsible sidebar. Editors can drag Weaves from the tree directly onto the calendar to schedule them.
 
[weave calendar]
[weave calendar outliner="true"]

9.5 List Shortcode

[weave list]

Renders a filterable, searchable list of Weaves. Results update instantly via AJAX — no page reload required.

AttributeDefaultDescription
typesall typesComma-separated type slugs to restrict the list to (e.g. types="task,project").
scopeallall — search all Weaves.
children — only direct children of the current post.
searchfalseSet to true to show a live search bar.
filtersnoneComma-separated filter specs (see below).
orderbytitletitle, date, or modified.
orderASCASC or DESC.
limit50Maximum results. Use -1 for all.

Filter specs

The filters attribute accepts a comma-separated list of specs. Each spec adds one filter control to the collapsible filter panel:

SpecFilter UIDescription
typeDropdownFilter by Type.
status:slugDropdownFilter by a specific status field (replace slug with the status field slug).
field:slugDepends on field typeFilter by a custom field. Text fields → text input; number/date fields → min–max range; select fields → dropdown; checkbox fields → Yes/No dropdown.
relation:slugHas / Has not dropdownFilter by whether a Weave has any relationship of the given type.
 
[weave list]
[weave list types="task" search="true"]
[weave list scope="children" orderby="date" order="DESC" limit="20"]
[weave list types="task,project" search="true"
    filters="type,status:task-status,field:due-date,relation:references"]

9.6 Editing & Action Shortcodes

[weave create]

Renders a Create button. Clicking it opens a modal where the user can enter a title and optional type. The new Weave is saved as a draft via AJAX. If placed on a Weave page, the new Weave is automatically added as a child, and the Outliner on the same page refreshes. Only visible to users with publish_posts capability.

AttributeDefaultDescription
labelCreateButton label text.
 
[weave create]
[weave create label="Add Task"]

[weave toolbar]

Renders a small icon toolbar for logged-in editors. Only visible to users with edit_posts. All actions save via AJAX without a page reload.

AttributeDefaultDescription
iconsarchive,favourite,hideComma-separated list of icons to show. Available: archive (toggle archived status), favourite (star), hide (hide/show on Kanban).
idcurrent postID of the Weave to act on.
 
[weave toolbar]
[weave toolbar icons="favourite,hide"]

[weave archive_button]

A dedicated Archive / Unarchive button for the current Weave. Clicking it toggles the archived status and reloads the page. Only visible to users with edit_posts.

AttributeDefaultDescription
labelArchive / UnarchiveCustom button label (overrides the automatic archive/unarchive text).
idcurrent postID of the Weave to act on.
 
[weave archive_button]
[weave archive_button label="Send to Archive"]

10. Settings Reference

All plugin settings are found under Weaves → Settings in the WordPress admin. The settings page is divided into tabs:

TabDescription
TypesCreate, edit, reorder, and delete Weave types. Set name, slug, colour, and Font Awesome icon for each. System types can be customised but not deleted.
StatusesCreate status fields (e.g. Task Status) and their options (e.g. To Do, In Progress, Done). Assign status fields to types and mark one as the main field per type. Optionally set a globally-default status field.
FieldsCreate custom fields with a name, slug, field type, and optional type scope. Reorder fields with drag and drop.
RelationshipsCreate named relationship types (e.g. References, Inspired By). Each has a name and slug.
